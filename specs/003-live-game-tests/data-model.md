# Data Model: 003-live-game-tests

**Date**: 2026-04-05

## Entities

### EngineFixture

Manages the lifecycle of a headless BAR engine instance shared across all integration tests.

**State transitions**: `Idle → Starting → Connected → Running → Disposed`

| Field | Type | Description |
|-------|------|-------------|
| client | BarClient | The FSBar.Client instance managing engine communication |
| initialFrames | GameFrame list | Warm-up frames captured during initialization (first 30 frames) |
| initialEvents | GameEvent list | Flattened events from warm-up frames (Init, UnitCreated, UnitFinished, etc.) |
| sessionDir | string | Temp directory for engine logs and artifacts |

**Lifecycle**:
1. `InitializeAsync` — creates BarClient with default headless config, calls `Start()`, runs 30 warm-up frames via `Step()`, captures frames and events
2. Tests use `client.StepWith()` / `client.Run()` to interact with the engine
3. `DisposeAsync` — calls `client.Stop()` which handles engine shutdown and cleanup

### Engine Prerequisites

Validated by `tests/check-prerequisites.sh` before engine-dependent tests run.

| Check | Source | Failure behavior |
|-------|--------|-----------------|
| Engine binary (spring-headless) | PATH or HIGHBAR_TEST_ENGINE env var | Skip integration tests |
| SPRING_DATADIR | Auto-detect from engine binary grandparent | Skip integration tests |
| Game archive | SPRING_DATADIR/packages/ | Skip integration tests |
| Map file | SPRING_DATADIR/maps/ | Skip integration tests |

### Test Report

Generated by `tests/run-all.sh` to `reports/testreports/`.

| Field | Type | Description |
|-------|------|-------------|
| timestamp | string | ISO-ish timestamp for the report file name |
| category | string | Which test category was requested |
| tierResults | map | Per-tier pass/fail/skip counts |
| environment | record | Engine path, version, game, map, platform |
